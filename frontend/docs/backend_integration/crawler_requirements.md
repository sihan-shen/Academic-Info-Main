# 爬虫数据需求说明文档

## 1. 概述
本项目旨在为学生提供全面的导师信息查询与匹配服务。爬虫的主要任务是从各大高校官网、院系主页及学术数据库中抓取导师的详细信息，构建结构化的导师数据库。

## 2. 核心实体：导师 (Tutor)
以下字段为必须抓取的核心数据，用于展示导师详情页及列表页。

### 2.1 基本信息 (Basic Info)
| 字段名 | 类型 | 说明 | 示例 | 来源建议 |
| :--- | :--- | :--- | :--- | :--- |
| `name` | String | 导师姓名 | "张明华" | 教师主页标题 |
| `avatar` | String (URL) | 头像链接 | "http://.../zhang.jpg" | 教师主页图片 |
| `school` | String | 所属学校 | "清华大学" | 网站域名/面包屑 |
| `department` | String | 所属院系 | "计算机科学与技术系" | 网站面包屑/部门列表 |
| `title` | String | 职称/头衔 | "教授" | 个人简介首段 |
| `tags` | Array<String> | 荣誉标签 | ["博导", "院士", "杰青"] | 个人简介/荣誉列表 |
| `email` | String | 联系邮箱 | "zhang@tsinghua.edu.cn" | 联系方式区域 |
| `phone` | String | 联系电话 | "010-12345678" | (选填) 联系方式区域 |
| `personal_page` | String (URL) | 个人主页链接 | "http://..." | 原始抓取URL |

### 2.2 详细介绍 (Details)
| 字段名 | 类型 | 说明 | 示例 | 来源建议 |
| :--- | :--- | :--- | :--- | :--- |
| `bio` | String (Text) | 个人简介 | "张明华，清华大学..." | "个人简介"栏目 |
| `direction` | String | 研究方向 | "人工智能、机器学习" | "研究方向"栏目 |
| `achievements` | String (Text) | 学术成果综述 | "发表顶级论文150+篇..." | "学术成果"栏目 |
| `guidance` | String (Text) | 学生培养情况 | "已指导博士生30余名..." | "教学情况"栏目 |
| `service` | String (Text) | 社会兼职/服务 | "中国人工智能学会副理事长..." | "社会兼职"栏目 |

### 2.3 结构化列表数据 (List Data)
以下数据需尽可能解析为结构化数组，而非纯文本。

#### A. 成长路径 (Growth Path / Education & Work)
*   **目标字段**: `growthPath`
*   **结构**: `[{ year: "2010", content: "获博士学位", type: "教育经历" }]`
*   **来源**: "教育经历"、"工作经历" 栏目。

#### B. 代表性论文 (Papers)
*   **目标字段**: `papers`
*   **结构**: `[{ title: "Deep Learning...", journal: "CVPR", year: "2016" }]`
*   **来源**: "发表论文"、"代表性著作" 栏目。需提取年份和期刊名。

#### C. 科研项目 (Projects)
*   **目标字段**: `projects`
*   **结构**: `[{ title: "国家重点研发计划...", role: "负责人", desc: "..." }]`
*   **来源**: "科研项目"、"在研项目" 栏目。

#### D. 合作资源 (Cooperation - 进阶)
*   **目标字段**: `coops`
*   **结构**: `[{ name: "Google AI Lab", type: "企业合作", desc: "..." }]`
*   **来源**: 文本分析，提取"与...合作"、"联合实验室"等关键词。

#### E. 学生去向 (Students - 进阶)
*   **目标字段**: `students`
*   **结构**: `[{ name: "李某", year: "2018", dest: "阿里达摩院" }]`
*   **来源**: "毕业生去向"、"校友风采" 栏目。

#### F. 社会关系 (Socials - 进阶) <!-- Added based on frontend usage -->
*   **目标字段**: `socials`
*   **结构**: `[{ role: "副理事长", org: "中国人工智能学会" }]`
*   **来源**: 解析 `service` 文本或 "社会兼职" 列表。

#### G. 风险排查 (Risks - 衍生数据) <!-- Added based on frontend usage -->
*   **目标字段**: `risks`
*   **结构**: `[{ type: "低风险", content: "近三年无学术不端记录" }]`
*   **来源**: 基于爬取内容分析（如学术不端记录库匹配）或人工录入。

## 3. 辅助实体：学校与学院 (School & Department)
用于构建筛选体系。

*   **学校列表**: 需包含 `name` (学校名), `location` (所在城市/省份), `level` (985/211/双一流)。
*   **学院列表**: 需包含 `name` (学院名), `school_id` (所属学校)。

## 4. 数据更新策略
*   **频率**: 建议每周或每月更新一次，以获取最新的论文发表和项目信息。
*   **增量更新**: 对比 `name` + `school` + `department`，若存在则更新字段，不存在则新增。

## 5. 特殊处理
*   **反爬虫策略**: 许多高校网站有简单的反爬机制，需注意 User-Agent 设置及访问频率。
*   **数据清洗**:
    *   去除 HTML 标签。
    *   统一日期格式 (YYYY-MM)。
    *   统一职称名称 (如 "Professor" -> "教授")。
